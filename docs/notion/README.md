# Notion Documentation Workflow

Use Notion MCP for documentation, package inventory, engineering notes, and progress tracking.

## Databases

1. **Documentation**: architecture docs, PRDs, runbooks, API docs.
2. **Packages**: package ownership, dependency risk, changelog.
3. **Notes**: investigation notes, design decisions, meeting notes.
4. **Progress**: task checkpoints, release status, verification outcomes.

When Notion MCP is unavailable, write draft entries under `docs/` and sync them later.
