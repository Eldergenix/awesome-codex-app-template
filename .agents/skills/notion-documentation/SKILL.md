## Instructions

Use the Notion MCP server when connected. Maintain four documentation surfaces:

1. Documentation database: architecture, PRDs, API docs, runbooks.
2. Packages database: package ownership, dependencies, risk, changelog.
3. Notes database: decisions, investigation notes, meeting notes.
4. Progress database: task status, checkpoints, verification results.

When MCP is unavailable, update `docs/notion/` and `docs/progress/` with a draft that can be synced later.
